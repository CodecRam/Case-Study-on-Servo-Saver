  • 4.1 Station details fail to load → cached details + warning.

Class Diagram & CRC Cards

FuelStation and FuelPrice anchor the legacy structure; PriceAlert and AlertNotification extend it without altering any existing class.
FuelStation core
Attributes: stationId, name, brand, openingHours, address, isOperating

Composes: FuelPrice (1→0..*), DailyPriceCap (1→0..*)
Aggregated by: FuelRetailer

Rule: at most one active FuelPrice per FuelType.
FuelPrice core
Attributes: priceId, amount, submittedAt, status

Validates against: DailyPriceCap
Part of: FuelStation (composition)

Rule: amount must not exceed the locked cap.
PriceAlert new
Attributes: alertId, fuelCode, thresholdPrice, radiusKm, quietHours, status

Belongs to: RegisteredAccountHolder
Matched against: FuelPrice

Rule: fires only when Active price ≤ threshold, outside quiet hours.
AlertNotification new
Attributes: notificationId, channel, sentAt, deliveryStatus

Part of: PriceAlert (composition)

Rule: failed deliveries retry up to 3 times.

State Machine — the FuelPrice Object

Full lifecycle of a single FuelPrice, from submission to destruction.
● Start→ Submitted→ Validating (format ‖ cap)→ Live (Active ⇄ Stale)→ Archived→ ◎ Final (7yr retention)

A terminate marker purges a price immediately if its station is deregistered. The moment a price becomes Active is the exact event Price Drop Alerts listens for.

User Interface Evaluation

Hands-on assessment of the live app's map view, list view, fuel-type selector and brand filter.
Good4
  • Map-first, colour-coded pins with "Lowest" badge
  • "Tomorrow's price cap" panel
  • Comprehensive filters / sort
  • Inherits SV trust & single sign-in
Needs work4
  • Entirely pull-based, no proactive alerts
  • No save / watchlist capability
  • App-wide timestamp, not per-station
  • Sort options not combinable
Defects2
  • "Opening hours not available" reads as broken
  • Inconsistent brand-filter logos

Proposed Improvement — Price Drop Alerts

Registered holders set fuel type, target price, radius and quiet hours; the system notifies them when a nearby station drops to target — reusing existing location, cap and account components.
